# Personal OS — תוכנית שדרוג מתועדפת (זורו)

הערכת מוכנות נוכחית: **~7/10** ככלי אישי. רחב ועובד, אך שביר (מספר מרכיבי-ליבה נשברו בשקט).
היעד: להפוך אותו ליציב, אמין ופרואקטיבי.

הסדר נקבע לפי **סיכון מול תועלת**: קודם מונעים נפילות שקטות ואיבוד דאטה, אחר כך יכולות.

---

## טכנולוגי

### P1 · רשת ביטחון לפריסה (CI)  — ✅ בוצע
GitHub Actions שרץ על כל push: `node --check` לכל פונקציה, ולידציית `vercel.json`,
ספירת פונקציות ≤12, בדיקת קבצים קטועים/NUL, ובדיקת `index.html`/`voice.js`.
*כבר תפס באג אמיתי: `sw.js` שנקטע — ותוקן.*

### P2 · אמינות דאטה + גיבוי  — הכי קריטי
מעבר מ-localStorage בלבד לשכבת אחסון אמיתית (Vercel KV / Upstash): סנכרון דו-כיווני,
גיבוי אוטומטי, ושחזור בין מכשירים. *דורש הפעלת KV ב-Vercel (אדריך).*

### P3 · ניטור שגיאות + התראה
מעקב שגיאות אמיתי (beacon ל-`/api/log-error` + תצוגת שגיאות; אופציונלי Sentry),
כדי שלא יקרה שוב 500 שקט.

### P4 · Router אחד לפונקציות
איחוד endpoints ל-`/api/[...]` עם ניתוב פנימי — משחרר מקום מתחת לתקרת 12 הפונקציות.

### P7 · קול חכם יותר + TTS איכותי
לטנציה נמוכה (streaming), קול עברי איכותי (ElevenLabs/Azure במקום קול הדפדפן),
ביטול-הד אקוסטי ל-barge-in קולי. *דורש מפתח TTS.*

### P9 · פיצול index.html + build step
פיצול קובץ 456KB למודולים — מקטין סיכון השחתה ומשפר תחזוקה. אחרון (גדול, פחות דחוף).

## פיצ'רי

### P5 · עוזר פרואקטיבי + push אמיתי
בריפינג בוקר, סיכום ערב, דחיפת דדליינים מהמודול לימודים — מתוזמן ונשלח.
הפעלת Web Push (VAPID) על תשתית `sw.js` הקיימת. *דורש מפתחות VAPID.*

### P6 · צייד משרות מלא-מעגל
מעקב הגשות, תזכורות פולואו-אפ, הכנה לראיון צמודה למשרה, ודה-דופ חכם בין ריצות.

### P8 · זיכרון/פרופיל מאוחד
פרופיל והקשר משותפים לכל הסוכנים (היום מפוצל) — מבוסס על שכבת הדאטה מ-P2.
