# Voice Hotspot — פרויקט נפרד (Hub)
> פרויקט עצמאי, נפרד מזורו/Personal OS. כאן מרוכז כל מה שעשינו: מפרטים, פרומפטים לקלוד‑קוד, מחקר, וסטטוס.
> הקוד עצמו נבנה ב‑Claude Code בתיקייה `C:\Users\user\voice-hotspot` (לא בריפו הזה). רץ מקומית ב‑http://localhost:3000.

## מה זה
קו‑פיילוט חי לשיחות מכירה/פגישות עסקיות: מאזין לשיחה, מבין את האדם שמהצד השני מהמילים + מהקול + מהשילוב, קולט דקויות שאדם מפספס, ונותן הכוונת סגירה בזמן אמת. בעתיד: רב‑תחומי (ראיונות, חקירות, כללי) לבחירת המשתמש.

## 3 הפונקציות
1. **ניתוח כפול** — קול (פרוזודיה) + מילים (NLP) → קריאה מאוחדת + סימון סתירה.
2. **הפרדת דוברים + בחירת מטרה** — צבע/שם לכל דובר, קישור שם, מיקוד באתה/לקוח/כולם.
3. **פרומפטר חי** — תמלול כבועות צבועות + קלף AI DECODE צבוע‑רגש + הכוונה מסתגלת.

## סטטוס נוכחי (מה עובד)
- שפת עיצוב "Signal" + זרימת דפים (הכנה → פרומפטר חי → סיכום). אומת בצילומים.
- 3 הפונקציות עובדות בדמו; 3 שכבות אימון (טקסט/קול/משולב).
- **דיוק קול** מכויל על RAVDESS: עוררות ~82% (F1 85%), קוטביות ~67% — speaker‑independent, לא overfit.
- **דיוק טקסט (NLP)**: מנוע מקומי VADER ~49% ← LLM (Gemini עם הקשר) **~63%**, וזיהוי שלילי קפץ 17%→52%.
- ספקי מנוע: Gemini (חינם) + מנוע מקומי; **Groq בהוספה** כגיבוי חינם; Claude אופציונלי בתשלום.
- מצב חינם מלא (Web Speech + Web Audio + מנוע מקומי) ללא תלות במכסה.

## השלבים הבאים
1. הוסף Groq (גיבוי חינם, פחות תלות במכסה).
2. הרצת LLM eval רחבה יותר (30–40 דיאלוגים) למספר יציב.
3. שלב 3 → שלב 1 (מודל קול עמוק → 88%+).
4. הרחבת טקסונומיה (רגשות בדידים + מיקרו‑רמזים) + רב‑תחומיות.

## הערה אחראית
"אמת/שקר" לא מזוהה מקול בצורה אמינה — מציגים **סימני לחץ/חוסר‑עקביות** כאיתות הסתברותי בלבד, לא הכרעה. זהירות יתר בתחומים רגישים (חקירות).
