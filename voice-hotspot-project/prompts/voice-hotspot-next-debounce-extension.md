# Voice Hotspot — המשך: debounce + תוסף Chrome ל‑Meet ו‑Zoom
> להעתיק לקלוד‑קוד. עבוד בשלבים, עם אימות וצילומים. שאל אותי בעברית עם אופציות בחירה אם צריך החלטה.

## 1) Debounce/דילול קריאות האימון (שלא נגיע ל‑429 בחינם)
- הוסף debounce לקריאות ה‑coaching: אל תקרא ל‑LLM בכל תור — אגד תורים בחלון של ~1.5–2.5 שניות ושלח קריאה אחת מסכמת.
- דלג על קריאה כשאין שינוי משמעותי ברגש/בכוונה מהתור הקודם (סף שינוי).
- תקרת קצב: לכל היותר ~1 קריאת אימון ל‑3–4 שניות; אם הגענו לתקרה, השתמש בקריאה האחרונה/מנוע מקומי.
- שמור על ה‑fallback הקיים (retry → מנוע חוקים מקומי) בלי שינוי.
- אמת: הרץ דמו מלא ווודא שאין 429, והאימון עדיין מופיע בקצב סביר.

## 2) חיווט אודיו לתוסף Chrome — ל‑Google Meet וגם ל‑Zoom (web)
- ממש לכידת אודיו אמיתית של הטאב דרך `chrome.tabCapture` (manifest v3), והזרם אותו לצינור ה‑STT/פרוזודיה הקיים (אותו pipeline של מצב חינם — Web Speech + Web Audio).
- **Google Meet** (`meet.google.com`): content script שמזהה שיחה פעילה, מציג overlay של הפרומפטר (בסגנון Signal), ולוכד את אודיו הטאב.
- **Zoom web** (`*.zoom.us` / web client): אותו דבר — overlay + לכידת אודיו של הטאב.
- אם המשתמש מדבר דרך המיקרופון, שלב גם אותו (mic + tab) כך שגם הצד שלך וגם הצד השני נכנסים לדיאריזציה.
- ה‑overlay: ניתן לגרירה/מינימום, לא חוסם את השיחה, תואם שפת Signal.
- הערות מציאות: לכידת אודיו של משתתפים מרוחקים תלויה בכך שהאודיו יוצא דרך הטאב; אם משהו לא נתפס, ציין זאת ופרט מה כן/לא עובד. (Recall.ai לנייטיב — בהמשך, לא עכשיו.)
- אמת: התקנת התוסף, overlay מופיע ב‑Meet וב‑Zoom‑web, ואודיו אמיתי זורם לתמלול+פרוזודיה. צילומים לכל אחד.
