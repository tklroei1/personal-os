# Voice Hotspot — תוספת לתוכנית: מסלול ללא מנועי LLM
> כל מה שאפשר להתקדם בו עכשיו בלי Gemini / Groq / Anthropic (בלי תלות במכסות). הכול מקומי. עבוד בשלבים עם אימות, מספרי דיוק וצילומים. שאל בעברית עם אופציות בחירה כשצריך החלטה.

## למה המסלול הזה
מנועי ה‑LLM נדרשים בעיקר ל**ניסוח** ההכוונה ולכיול שכבת הטקסט. אבל חלק גדול מהמוצר — הקול, האותות, הדינמיקה, ה‑UI, התשתית — לא צריך LLM בכלל. נתקדם בכל אלה עכשיו, ונשאיר את קריאות ה‑LLM ל"וו" אחד מסודר שיתחבר כשהמכסה תתחדש.

---

## 1) מודל הקול — ללא LLM (עבודת ליבה)
- שילוב מודל העוררות המכויל (~82%) ל‑`prosody.js` (שלב 3) — אם עוד לא הושלם.
- התקדמות למודל הקול העמוק (wav2vec2/HuBERT) — **inference מקומי, לא דורש API**. נרמול דובר, יעד 88–92%.
- הרחבת פלט הקול לרגשות בדידים שניתן להפיק מאודיו: התרגשות, היסוס, לחץ, שעמום, חום/קור רגשי — מאומן/מכויל מקומית.

## 2) מיקרו‑רמזים ואותות — חישוב מקומי טהור (בלי LLM)
זה הזהב שאדם מפספס בשיחה, והכול ניתן לחישוב מהאות עצמו:
- micro‑pauses לפני "כן"/לפני תשובה, השהיות חריגות.
- שינויי גובה צליל פתאומיים, עליית pitch בשאלה.
- האטה/האצה בקצב, שינוי באורך משפטים, ירידת אנרגיה הדרגתית.
- שינוי בעוצמה, רעידות, יחס דיבור/שתיקה.
- סמן כל אות עם חותמת זמן והסבר קצר ("האטה כשעלה המחיר").

## 3) דינמיקת שיחה — מקומי (בלי LLM)
- מי מוביל, יחס דיבור/הקשבה לכל דובר, קטיעות, זמני תגובה.
- "מדד מומנטום סגירה" חי — ציון משוקלל מכל האותות המקומיים (בלי LLM), עם מגמה (משתפר/מתדרדר).
- ציר זמן רגשי לכל השיחה.

## 4) שכבת חוקים להכוונה — מקומית (בלי LLM)
- שפר את מנוע החוקים המקומי הקיים (ה‑fallback) שייתן הכוונת מכירה טובה גם בלי LLM: מיפוי אות→המלצה (למשל "זוהה היסוס אחרי מחיר → הצע פריסת תשלום").
- ספריית התנגדויות + תגובות מוכנות (SPIN/Sandler) כטקסט קבוע, נבחר לפי האות שזוהה.
- ככה המוצר נותן ערך אמיתי גם במצב חינם מוחלט, וה‑LLM רק "מלטש" כשזמין.

## 5) הפרדת דוברים וקישור שם — מקומי (בלי LLM)
- שיפור הדיאריזציה ההיוריסטית (אשכולות pitch/energy + שתיקות).
- חיזוק `name-binding` (עוד תבניות בעברית/אנגלית, תיקון ידני נוח).

## 6) UI / UX — בלי LLM (הליטוש שביקשת)
- רספונסיביות מלאה למובייל.
- אימות שהטוגלים (Add details / קבוצות הקונטקסט) מסתירים/מציגים נכון.
- מיקרו‑אנימציות עדינות (reduced-motion מכובד).
- מצב סקירה אחרי שיחה: ציר זמן רגשי, רגעי מפנה, "מה פספסת" — מרונדר מהאותות המקומיים.
- ליטוש הפרומפטר: 3 השכבות ברורות, מיקרו‑רמזים מודגשים בעדינות, "המהלך הבא" בולט.

## 7) Domain Packs — מבנה וחוקים, בלי LLM
- בנה את התשתית: בורר תחום + "domain pack" (טקסונומיה + חוקי הכוונה + ניסוחים קבועים).
- מלא את חבילת **מכירות** במלואה כחוקים מקומיים. שלד לראיונות/חקירות/כללי.

## 8) תוסף Chrome — לכידת אודיו, בלי LLM
- חיווט `chrome.tabCapture` ל‑Google Meet ול‑Zoom‑web → צינור ה‑STT/פרוזודיה המקומי.
- overlay בסגנון Signal, גריר/מינימום. (אין כאן צורך ב‑LLM.)

## 9) תשתית, דאטה, eval — בלי LLM
- צינור פיצ'רים, cache, harness, הכנת דאטהסטים, דוחות דיוק — הכול מקומי.
- הכנת ה"וו" היחיד לקריאות LLM (ממשק נקי) כך שכש‑Gemini/Groq/Claude יחזרו — מתחברים בשורה אחת.

---

## סדר מוצע (בלי LLM)
1. ליטוש UI/UX (#6) — מיידי, מה שביקשת.
2. מיקרו‑רמזים + דינמיקה + מומנטום סגירה (#2,#3) — הכי משדרג ערך, מקומי לגמרי.
3. שכבת חוקים להכוונה מקומית (#4) — שהמוצר שימושי גם בלי מנוע.
4. מודל קול עמוק (#1) — הדרך ל‑88%+.
5. domain packs מקומי (#7) + תוסף Chrome (#8).
6. תשתית/eval (#9) לאורך הדרך.

## כשהמנועים יחזרו (Gemini/Groq/Claude) — נפרד
- כיול שכבת הטקסט, ניסוח ההכוונה החי, ה‑ensemble/ראוטר. עד אז — לא תלויים בהם.
