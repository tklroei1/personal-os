# Voice Hotspot — פרומפט מלא לקלוד‑קוד (להעתיק כמו שהוא)
> מחליף את כל ההנחיות הקודמות על UI/מבנה. עבוד לפי הסדר.

## ⚠️ קודם כול — סקילים ושיטת עבודה (אל תדלג, זה לא נעשה קודם)
1. התקן את הסקיל **frontend-design** לתוך הפרויקט בנתיב `.claude/skills/frontend-design/SKILL.md` (מקור: github.com/anthropics/skills). התקן גם את **obra/superpowers**. אם הסקיל חסר — עצור ובקש אותו ממני.
2. אשר בתשובה הראשונה שהסקיל קיים ושאתה משתמש בו (ציין את הפלטה/הטיפוגרפיה/הלייאאוט שהוא הוליד).
3. **קודם עיצוב, אחר כך קוד:** לפני שאתה נוגע בקוד UI, הצג לי תוכנית עיצוב — פלטה חמה (5–7 צבעים עם שמות והקס), זיווג פונטים, רעיון לייאאוט, ואלמנט חתימה אחד — קבל ממני אישור, ורק אז תבנה.
4. **שאלות אליי:** מהיום, בכל החלטה שאתה צריך — שאל אותי **בעברית** עם **אופציות בחירה** שאני בוחר מהן. שמור על הסגנון הזה עד שאגיד אחרת.

## 🆓 מצב חינם — בנה את זה קודם, שיהיה שימושי לגמרי
הרץ הכול על רכיבים חינמיים, עם מסלול שדרוג נקי כשיתווספו מפתחות בתשלום:
- **תמלול חי:** Web Speech API בדפדפן (חינם, עברית + אנגלית).
- **רובד קולי בלי Hume:** חילוץ פרוזודיה בדפדפן עם Web Audio API — גובה צליל, עוצמה, קצב דיבור, הפסקות/אנרגיה → סימני לחץ/עוררות.
- **הפרדת דוברים בלי Deepgram:** דיאריזציה היוריסטית (אשכולות לפי גובה/אנרגיה + פערי שתיקה) + **קישור שם** מ"קוראים לי X", עם תיקון/תיוג ידני של המשתמש.
- **רגש מהמילים + הכוונה:** Gemini 2.5 Flash בחינם (המפתח כבר ב‑.env) דרך הראוטר הקיים.
- **מסלול שדרוג:** עטוף STT, דיאריזציה, ורגש‑קול מאחורי ממשקים, כך שהוספת `DEEPGRAM_API_KEY` / `HUME_API_KEY` בעתיד מחליפה את המודול החינמי במודול בתשלום **בלי לכתוב מחדש**. כשאין מפתח בתשלום — דילוג אלגנטי + תווית קטנה "מצב חינם".

## המוצר = 3 פונקציות מודולריות (כל אחת ברורה ונפרדת)

### פונקציה 1 — ניתוח רגש כפול (קול + מילים → קריאה אחת מאוחדת)
- **מהקול עצמו:** רעידות, קצב דיבור (מהיר/איטי), גובה צליל, עוצמה, הפסקות/היסוס, אנרגיה → מצב רגשי.
- **מהמילים (תמלול):** רגש + כוונה דרך LLM.
- **איחוד שניהם** לקריאה אחת של מה האדם מרגיש/רוצה; **סימון סתירה** בין הטון למילים. פלט לכל תור: תווית רגש + רמת ביטחון + "למה" קצר (אילו רמזים).

### פונקציה 2 — הפרדת דוברים + בחירת מטרה
- זיהוי הקולות השונים בשיחה.
- **קישור שם:** מישהו אומר "שלום קוראים לי דנה" → הקול מתויג "דנה" וממשיך כך.
- **בחירת מטרה:** המשתמש בוחר את מי ה‑AI מנתח — אדם אחד / רק הלקוח / כולם (2+). ניתן להחלפה תוך כדי שיחה. רק המטרה הנבחרת מקבלת ניתוח מעמיק + הכוונה.

### פונקציה 3 — הפרומפטר (פאנל חי בצד, עובד כתוסף)
- פאנל קריא ורגוע בצד השיחה שמציג:
  - **תמלול כבועות צ'אט** — שם הדובר בכל תור, ו**צבע שונה לכל מתקשר**.
  - **בועות ה‑AI בעיצוב וצבע נפרדים** לגמרי מהתמלול. צבע בועת ה‑AI יכול להתאים לרגש שהיא מעבירה (כעס/דחיפות = אדום, חיובי = ירוק).
  - לכל תור: **פירוש + הסבר**, למשל "התחיל לדבר מהר → ייתכן לחץ".
- **הכוונת סגירת מכירה חיה ומסתגלת:** כשמזוהה למשל לחץ אחרי שהצעת X → "הלקוח נלחץ אחרי X — נסה Y כדי ש‑Z יקרה, ואז המשך ל‑W". מתעדכן כל הזמן לפי מהלך השיחה והניתוח.
- **קצב קריאה:** נוח וקריא — קצב חשיפה מבוקר, בלי להציף את המסך.

## UI / מבנה — בהירות היא התיקון מספר 1
- **הפרדת דפים ברורה (מסלול אמיתי, לא מסך אחד צפוף):**
  1. **הכנה לשיחה ספציפית** (דף): מה אתה מוכר, הצעות ערך, התנגדויות נפוצות, מטרת השיחה, ובחירת הדובר/ים למיקוד.
  2. **שיחה חיה + פרומפטר** (דף נפרד / פאנל התוסף): בועות תמלול + בועות הכוונת AI + מד הרגש הכפול.
  - ניווט ברור בין השניים.
- **עיצוב:** גדול יותר, חם יותר, יפה ונעים לעיניים, קריאוּת גבוהה. **הפרדת צבעים חזקה** (צבע לכל דובר; צבע ה‑AI נבדל; בועות AI צבועות לפי רגש). לפי הסקיל frontend-design — לא דשבורד גנרי.

## תוסף / פלטפורמה
- בנה כך שיוכל לרוץ כ**תוסף שמתחבר ל‑Zoom ול‑Google Meet (עיקרי)**. MVP: **תוסף Chrome** שלוכד את אודיו הטאב של Meet/Zoom‑web דרך `chrome.tabCapture` ומציג מעליו את פאנל הפרומפטר. בהמשך: בוט Recall.ai לנייטיב.

## הכול חי
- אודיו אמיתי מהמיקרופון/שיחה → STT חי → רגש‑קול חי + רגש‑מילים חי → פרומפטר + הכוונה חיים.

## סדר העבודה
1. התקן ו**עקוב** אחרי הסקיל frontend-design → תוכנית עיצוב → הצג לי לאישור.
2. בנה מחדש את המבנה לזרימת 2 הדפים (הכנה → פרומפטר חי).
3. פונקציה 1 (ניתוח כפול) → פונקציה 2 (דיאריזציה + בחירת מטרה + קישור שם) → פונקציה 3 (פרומפטר: בועות צבועות לדובר + בועות AI נפרדות צבועות‑רגש + הסברים + הכוונה מסתגלת).
4. בדוק חי מקצה לקצה (אנגלית וגם עברית).
5. מסלול תוסף Chrome ללכידת אודיו של Meet/Zoom.
6. צילומי מסך + ביקורת עצמית מול הסקיל בכל שלב UI.
