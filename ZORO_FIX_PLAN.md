# תוכנית תיקון וניקוי — מערכת זורו (Personal OS)

תאריך: 2026-06-19 · נכתב לאחר בדיקה ממוקדת של מצב ה-git, הקבצים והפריסה.

---

## TL;DR

**החדשות הטובות:** האתר החי תקין ועדכני. ה-git המקומי (HEAD) מסונכרן עם `origin/main` ועם מה
שפרוס ב-Vercel — והגרסה הזו כוללת את כל השדרוגים (קול Whisper, push, גיבוי, איחוד פונקציות).

**הבעיה:** עץ העבודה המקומי (working tree) על הדיסק נתקע במצב **ישן והפוך**, וה-index של git
מבולגן. אם תעשה `commit` למצב הנוכחי — **תהרוס את האתר החי**: תחזיר גרסה ישנה, תשבור את הפריסה
(חריגה ממכסת 12 הפונקציות של Vercel Hobby), ותאבד את רשת הביטחון (CI).

**מה צריך:** ליישר את עץ העבודה ל-HEAD, למחוק קבצי זבל, למחוק ענף תקוע, להקשיח את `.gitignore`,
ולשמור את העבודה החדשה (voice-hotspot-project, zoro-docs). הכול עם גיבוי לפני כל פעולה.

## 1 · מה לא בסדר — אבחון מדויק

### 1.1 עץ העבודה הפוך לגרסה ישנה
`index.html` על הדיסק = **415,865 בייט, ללא קול Whisper** — בעוד שב-HEAD (החי) יושב
`index.html` = **475,157 בייט, עם Whisper**. `git diff HEAD` מראה פער של ~7,800 שורות.
אותו דבר חל על `vercel.json`, `sw.js`, `voice.js`, `api/claude.js` ועוד — כולם בגרסה ישנה על הדיסק.
> מסוכן: ה-blob של `index.html` שעל הדיסק **לא תואם אף קומיט בהיסטוריה** → מגובה לפני מחיקה.

### 1.2 ה-index של git מושחת (מצב "חצי-reset")
`git status` מציג קבצים גם כ-`D` (staged למחיקה) וגם כ-`??` (untracked) במקביל — סתירה
שמעידה על index פגום (כנראה סנכרון/checkout שהשתבש מול מכשיר אחר).

### 1.3 קבצי זבל שישברו את הפריסה
על הדיסק קיימים 3 קבצים שכבר אוחדו לתוך `api/ai.js` (קומיט 87251bb) כדי לרדת מתחת ל-12:
`api/gemini.js`, `api/job-coach.js`, `api/match-score.js`.
נוכחותם = **15 פונקציות** → פריסת Vercel תיכשל. הם צריכים להימחק.

### 1.4 קבצי גיבוי/ביניים מיותרים
`index.html.new` (גרסת ביניים), `index.html.prevdeploy.bak` (גיבוי ישן),
ו-3 קבצים זמניים שנוצרו בבדיקה: `_head_index.tmp`, `_head.txt`, `_disk.txt` (+`_deltest.txt`).

### 1.5 ענף-נעילה תקוע
`refs/heads/main.lock.stale_1781787330` → מצביע לקומיט ישן `cc52fc2`. ref זבל, צריך למחוק.

### 1.6 `.gitignore` חלש
מכיל רק `.vercel` ו-`.env*.local`. לא מתעלם מ-`node_modules/` (יש 50MB ב-whatsapp-bot),
ולא מקבצי `*.bak/*.new/*.tmp` — מה שאיפשר לבלגן הזה להיווצר מלכתחילה.

## 3 · התוכנית — שלב אחר שלב (בטוח, הפיך)

> כל שלב מאומת מול המצב האמיתי של הריפו. שלב 0 מבטיח שאפשר לחזור אחורה תמיד.

**שלב 0 — גיבוי מלא.** יצירת ארכיון ZIP של כל התיקייה (ללא node_modules) לפני נגיעה ב-git.

**שלב 1 — יישור עץ העבודה ל-HEAD.** `git reset --hard HEAD` → מחזיר את כל הקבצים המנוהלים
לגרסה החיה (Whisper, push, נתיבי גיבוי, CI, איחוד פונקציות). מבטל את ה-index המושחת.

**שלב 2 — מחיקת זבל untracked.** מחיקת `api/gemini.js`, `api/job-coach.js`,
`api/match-score.js`, `index.html.new`, `index.html.prevdeploy.bak`, וקבצי `_*` הזמניים.

**שלב 3 — מחיקת ענף-הנעילה התקוע** `main.lock.stale_1781787330`.

**שלב 4 — הקשחת `.gitignore`** (הוספת `node_modules/`, `*.bak`, `*.new`, `*.tmp`, `_*.tmp`).

**שלב 5 — אימוץ zoro-docs כקנוני.** הסרת כפילויות התיעוד מהשורש, שמירת `zoro-docs/`.

**שלב 6 — קומיט נקי** של העבודה החדשה (voice-hotspot-project, zoro-docs, _PROJECT_MAP,
ROADMAP, .gitignore מוקשח).

**שלב 7 — אימות (ודאות שזה תוקן):**
- `node --check` לכל `api/*.js` → חייב לעבור.
- ספירת פונקציות `api/*.js` ≤ 12.
- `vercel.json` = JSON תקין.
- `git status` נקי, `HEAD == origin/main` (האתר החי לא נפגע).
- הרצת `scripts/ci-validate.mjs` (רשת הביטחון מ-P1).

לסביבת העבודה שלי **אין הרשאת מחיקה** על התיקייה הזו ו**אין כתיבה ל-`.git`**
